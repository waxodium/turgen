# Contributing to Turgen Shell
> waxodium will be happy to see anyone joins in to work with.

Welcome! For new Turgen Shell Developers. 
- **Explore the Code**: If you are new, start by reading the [DOCUMENTATION](./DOCUMENTATION.md) to understand how the turgen shell source code works together.

- **Find an Issue**: Look through *Turgen Shell's* [Issue Tracker](https://github.com/waxodium/turgen/issues) for tasks labeled `good first issue` or `help wanted`.

- **Reach Out**: If you are unsure about an idea or need help understanding the architecture, please don't hesitate to [open a Discussion](https://github.com/waxodium/turgen/discussions/) or reach out to me directly. I am happy to chat and walk you through.

- **Create an Issue**: If you spot a flaw, or bugs. Please, may freely [Open an Issue](https://github.com/waxodium/turgen/issues) on turgen repository

## How to contribute?
There are **3** ways to contribute to the turgen repository.

### 1. Forks
- Make a fork of [turgen repository](https://github.com/waxodium/turgen) to your own personal contributing github account.
- Make changes to your forked repository. Be sure to read the [DOCUMENTATION](./DOCUMENTATION.md) when making changes.
- Contribute to turgen via submitting a Pull Request **(PR)** from your forked repository.

### 2. Reporting Issues or Discussion
If you aren't ready to write code yet, you can still contribute by:

- **Opening an Issue**: If you find a bug or have a suggestion, please open an issue in the [Issues tab](https://github.com/waxodium/turgen/issues).

- **Starting a Discussion**: If you have questions about the architecture or want to suggest a new feature, feel free to start a [discussion](https://github.com/waxodium/turgen/discussions).

### 3. Feature Branch
- Create a branch of the turgen repository by cloning it
```bash
git clone https://github.com/waxodium/turgen.git
```
or if you have an existing cloned repository. Please ensure that your local repository is up to date
```bash
git checkout main
git pull origin main
```

- Create your branch
```
git checkout -b branch
```
make appropiate changes by also following the turgen's [DOCUMENTATION](./DOCUMENTATION.md)

- Push your code and starts a pull request to be await for the review to merge


# Important!
If your changes involve significant changes to the codebase, please open an issue start a discussion first to reach out to the repository maintainer
